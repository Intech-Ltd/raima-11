Introduction

1.1 Overview

The RDM database management system (DBMS) is designed to provide powerful, flexible, high-performance capabilities for developing C language database applications. By combining the network and relational model technologies in a single system, RDM lets you organize and access information efficiently, regardless of the complexity of your data. This combined technology provides tremendous speed advantages, and minimizes data redundancy.

The RDM DBMS is written in C, so the applications you create with it are easily ported to a variety of operating systems, including Linux, UNIX, and Windows. The potential market for your applications spans from PDAs and PCs, to LANs and WANs. Since you have complete source code you have full control over any application you create.

The major features of RDM include:

  • Multi-user support for local-area networks (LANs), wide-area networks (WANs), and multi-tasking, multi-threaded operating systems.
  • Multiple configurations. Standalone, direct-link (multi-user) and client/server (multi-user).
  • Transaction processing and logging.
  • Automatic database crash recovery.
  • Access methods based on the network and relational database models.
  • A Database Definition Language patterned after C, for defining the content and structure of an RDM database.
  • Multiple database access. More than one database can be opened and accessed at a time from an application program.
  • Incremental opens and closes of multiple databases.
  • Interactive and batch database access utilities, for easy manipulation of the information in an RDM database.
  • File transfer utilities for importing and exporting data between ASCII text files and an RDM database.
  • A database consistency check utility for reporting any problems with database integrity.
  • Virtual memory disk caching, for fast database access.
  • A runtime library of over 150 functions, for complete database control.

1.2 RDM Documentation

The RDM online documentation is a multi-volume set. The book set is presented in HTML format and included with your product package. 

1.2.1 About This Manual

This User's Guide, contains general information and examples on the use of the system and is organized as described below.

Database Concepts should be read by everyone, regardless of the extent of their database background. It presents basic RDM database concepts, such as definitions of the database terminology used in this manual, a description of the network database model and the differences between it and the relational model, and an explanation of the elements of an RDM database.

Operational Overview  gives an overview of each of the components of an RDM DBMS. It describes the general operational flow for creating an RDM application program, and provides a simple (but complete) introductory example.

Database Design  describes in detail RDM's Database Definition Language (DDL) and the operation of the DDL processor. It also explains how to design an RDM database, and includes an example of database design that illustrates some of the more commonly used structures.

Database Manipulation explains the use of RDM runtime functions for controlling, accessing, and manipulating a database from a C application program.

Transaction Processing  describes transaction processing functions and tells you how to use transactions in a single- or multi-user environment.

Multi-User Overview  describes the use of RDM in multi-user, shared database applications. Basic multi-user concepts are presented, as is a detailed description of RDM's multi-user implementation. This also explains how to use the runtime multi-user control functions to efficiently synchronize shared database access.

Unicode Support describes the API that accepts UNICODE strings as parameters.

The section entitled Develop Platform Independent Applications documents the Platform Support Package (PSP) used internally in the RDM engine and utilities to achieve platform independence. RDM applications are free to utilize these same functions.
