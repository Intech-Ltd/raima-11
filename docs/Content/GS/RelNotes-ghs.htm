Green Hills INTEGRITY 10.0.x Platform Release Notes

Introduction  

This PS (Platform-Specific) Guide focuses on the INTEGRITY 10.0.x operating system using "Single Address Space POSIX" for RDM. This section explains:

  • Integrity
  • Creating an Integrity kernel
  • Building RDM for Integrity

RDM for Integrity is set up as a host/target install with a cross compiling procedure. Supported host platforms are Linux or Windows.

This release of RDM is supported for Green Hills INTEGRITY 10.0.x on x86.

INTEGRITY

INTEGRITY is a POSIX like operating system for the embedded space. An applications can use a number of address spaces with protection between them or they can be linked directly into the kernel. INTEGRITY also supports downloading of address spaces. Each address space is a fully linked (possibly with use of shared libraries between the address spaces). Applications must be put together using an integration tool before they can be downloaded to the target.

With INTEGRITY you have the option to use a real target or a simulator. The simulators are simulators that actually simulates the instruction set of the architecture. It is not an environment using you host instruction set with a BSP on top of your host OS to emulate the INTEGRITY environment.

With INTEGRITY there are different options on how to get your code running on a board or a simulator:

  • A kernel that is able to dynamically download and load applications. When you are done the application can be unloaded.
  • Compile your application together with the kernel using the kernel's address space.
  • Use a number of address spaces. E.g. One for the kernel, one for the virtual file system server, one for the TCPIP stack, and one for the application.

Our project files use the first approach using "Single Address Space POSIX".

Creating an INTEGRITY kernel

Configuring and compiling an INTEGRITY kernel for "Single Address Space POSIX" is done from within the Multi Project Manager. Make sure to include ffs, socket, net, and rpc.

Building RDM for INTEGRITY

For a Pentium target open the project file in the Multi Project Manager:

  target/integrity/pcx86/default.gpj

You may need to change the location of the INTEGRITY target OS to match your installation. Please edit the project file before opening it or set the build options inside the Multi Project Manager to reflect your installation.

There are some sub projects that are greyed out. These are not included in the package you have. They are either sub projects for internal tests and tools that are not distributed or sub projects that are not included in the particular package you have.

Build the project from within the Multi Project Manager.

Tools and examples will show up under target/integrity/pcx86/mod. The files with extension '.ael' are INTEGRITY applications you can load and run on the target. You will not be able to set program arguments so most of the tools will not be applicable for INTEGRITY.
