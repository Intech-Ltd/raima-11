Choosing The Right API

RDM provides several methods to interact with a database. The best API to use varies on your specific situation. The following is a brief summary of the API options in RDM.

The RDM Core-Level C API (Native d_)

The RDM database management system (DBMS) is designed to provide powerful, flexible, high-performance capabilities for developing C language database applications. All operations needed for complete database operations are available in its proprietary C application programming interface (API) runtime library of more than 150 functions. All of the other APIs provided in RDM (e.g., SQL) use the core API functions. Functions that perform database-related operations are prefixed with "d_".

Also provided are the Platform Support Package functions, prefixed with "psp_", that provide an operating system independent API to typical operating system functions such as dynamic memory management and multi-process synchronization. Data type and constant declarations and well as the prototypes for all of these functions are provided in the standard header file: rdm.h.

See the Native API Guide for more information.

The RDM C++ API

The C++ interface to RDM is designed to augment the RDM navigation API by providing database specific sets of classes implementing higher-level abstractions of the Core API. These generated interfaces are built to fit your specific database schema using an object orientated approach. The C++ API is designed for ease of use, but by giving full access to both RDM's network and relational functionality for RDM, it is very powerful and can be used to create efficient database applications. The C++ interface is the only API that implement a common interface with live cursors for table scans, key scans, and set member scans (only partial live).

See the C++ API Guide for more information.

The RDM Objective-C/Cocoa API

The Objective-C 2.0 interface to RDM is designed to augment the RDM navigation on Mac OS X and iOS platforms by creating custom objects to represent records and interface with the database. This combines the performance of RDM with the intuitiveness of an object oriented interface, making it easy to integrate with Cocoa applications.

See the Objective-C API Guide for more information.

The RDM SQL Native API

The RDM SQL native application programming interface (API) presents a small-footprint, efficient and easy-to-use C language interface for performing database operations using the RDM SQL language. The API has been designed to be similar to ODBC (Open Data Base Connectivity) but without the complexities that ODBC introduces due to the DBMS-independence the ODBC standard is intended to provide. As such, this API and associated SQL Language is proprietary and doesn’t conform to the ODBC standard. Applications written using this SQL API are directly linked to the RDM database engine.

All of the functions are prefixed with "rsql". Many use special data types provided in the standard header file: rsql.h.

See the RSQL API Guide for more information.

The RDM ODBC SQL API

RDM provides a separate ODBC API that allows SQL applications to be written using the industry-standard Open Data Base Connectivity C language interface to perform database operations using standard SQL. Applications written using this API can be both directly-linked to the RDM database engine or can run as client programs that access a RDM SQL server program (rdmsqlserver) running on a separate computer. Even though ODBC was developed by Microsoft for use on the Windows operating system, the RDM ODBC implementation is available on every OS platform that RDM supports.

All of the functions are prefixed with "SQL". ODBC data types and constant declarations are provided in the standard header files: sql.h for basic ODBC functionality; sqlext.h for basic and extended ODBC functionality, sqlrext.h for basic, extended, and RDM-specific functionality.

The RDM ODBC implementation is based on version 3.51 of the Microsoft ODBC specification.

See the ODBC API Guide for more information.
