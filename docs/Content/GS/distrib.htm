Runtime Redistributable Specification

After developing an application using the RDM development environment, deployment of the application to the target system will require the developer to also redistribute all the files that are required to support the application on the target system. The tables included in this section outline the required redistributables for applications using shared or dynamic libraries. Applications using static libraries can disregard this section of the manual.

Library Naming Convention

The library naming convention is as follows:

{prefix}{library name}{version}.{extension}
{prefix} UNIX/Linux releases use a "lib" prefix. Windows releases do not contain a prefix.
{library name} Reference the table below
{version} The version corresponds to the product release version.
{extension} UNIX/Linux releases will use either a ".so", ".sl", or ".dylib" extension depending on the operating system. Windows platforms will be ".dll".

An example of the platform support pack library (rdmpsp) naming is:

Windows:

            rdmpsp11.dll

UNIX/Linix:

librdmpsp11.so

Mac OS:

librdmpsp11.dylib

Static Libraries (selected cross-compiled platforms only and iOS):

librdmpsp11.a
