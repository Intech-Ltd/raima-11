QNX Neutrino Platform Release Notes

Installing & Compiling RDM for QNX Neutrino 6.5.x

RDM for Neutrino is set up as a host/target install with a cross compiling procedure. Supported host platforms are Linux or Windows with the QNX Neutrino 6.5.x SDP installed. Please refer to the QNX SDP system requirements for specific requirements of the host platform.

This release of RDM is supported for Neutrino 6.5.x on x86 and ARMv7 (Cortex Series).

There is a known issue on ARM platforms with the handling of de-normalized floating point values. It is advised not to use a floating point value in an index on ARM based platforms.

Install

Please see the Hosted Platform Install Guide for installation instructions.

Compile

  1. In a command shell, ensure that the following environment variables are set (they should already be automatically set in a default shell when the Neutrino SDP is installed):

QNX_HOST

QNX_TYPE

  1. From the top level of the RDM install directory, execute:

On Unix:

. ./rdm.sh -neu{arch} -{tfs}

Note, this command must be sourced, by prepending it with ". " as shown above

{arch} may be "x86" or "arm"

{tfs} may be "-tfsr", "-tfss", or "-tfst". All versions of the tfs are supported on Neutrino, including the tfserver.

On Windows:

rdme.bat neu{arch} {tfs}

{arch} may be "x86" or "arm"

{tfs} may be "-tfsr", "-tfss", or "-tfst". All versions of the tfs are supported on Neutrino, including the tfserver.
