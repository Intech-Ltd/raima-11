Microsoft Windows Platform Release Notes

Introduction

This PS (Platform-Specific) Guide focuses on the Microsoft Windows operating system for RDM. This section explains:

  • How to set up the Windows development platform
  • How to build RDM Source & Examples
  • Other issues specific to Windows

Development Environment

The build scripts for RDM examples and source assume that the paths and environment variables for the desired compiler have already been setup. The following table lists the compiler versions supported and the environment variable used to indicate which compiler is being used.

The environment variables listed below can be found in the RDM setup script files used for setting the needed environment variables for RDM compiling.

Visual Studio Project Files (VS2008 and VS2010) and CMake support have also been provided for building and running examples and tutorials. To access the VS Project files, look in the RDM shortcut menu under GettingStarted. For instructions using CMake, see here.

RDM Setup Script Files

rdm.bat [home] [DEBUG] [target] [tfs] [VERBOSE]

Parameters

home Root directory of the RDM directory structure. Defaults to the current directory.
[NO]DEBUG Specify Debug option- default is OFF
[NO] VERBOSE Specify output level for the setup script. Default is NO Verbose.

Target

32bit Specify the target platform architecture as 32bit x86.
64bit Specify the target platform architecture as 64bit x86.
vx6x86 Specify cross compile for WindRiver VxWorks on the Intel x86 Processor
intx86 Specify cross compile for Green Hills Integrity on the Intel x86 Processor
neux86 Specify cross compile for QNX Neutrino on the Intel x86 Processor
neuarm Specify cross compile for QNX Neutrino using ARMv7 instruction set (Cortex Series)

The default is based on the environment variable: 
%PROCESSOR_ARCHITECTURE%

TFS

tfsr Use the Client/Server TFS version
tfss Use the Standalone TFS version
tfst Use the Direct Linked TFS version (Default)

If using multiple shell windows, changing the current TFS will affect all shells using that install directory.

Building RDM (source package) or Examples

  1. Install the software following the Windows Install Guide. If building source, ensure you are installing the source package.
  2. From a terminal window session:
>cd [installDir]
>rdm.bat [options]
>k

As part of the build step (for source or examples), a tfserver is automatically started before and stopped after. Shutdown any running tfservers before starting the build.

If a source package is installed, this will build the entire source if source files are modified. If source is not present, only the examples will be built.

All RDM makefiles are designed to be invoked by the KMake utility, which is included in the RDM Installation. The correct command is aliased to 'k' for convenience.

NOTE: When compiling with Kmake on Windows, the "rc.exe" tool is required. This can be obtained through the Windows Platform SDK.
