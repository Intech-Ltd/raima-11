rsqlShowPlan

Show a query's execution plan as a result set

Prototype

RSQL_ERRCODE EXTERNAL_FCN rsqlShowPlan(
    HSTMT       hPlan,  
    HSTMT       hQuery)

Arguments

hPlan (input) Statement handle which will contain the result set.
hQuery (input) Statement handle of query whose execution plan is to be shown.

Description

Call rsqlShowPlan to return a result set that shows the execution of a compiled query. The result set will be available through the hPlan statement handle so that upon successful return from a call to this function you can call rsqlFetch to retrieve the execution plan for the select statement that has been compiled on statement handle hQuery. The result set will contain one row for each table specified in the from clause of the select statement associated with hQuery. Each column of the result set is described in the following table.

Column
Number		 Column
Name		 Type Description
1 stepno smallint The plan step number, beginning with 1. The tables are accessed in this order.
2 catalog char(32) The name of the catalog associated with the database.
3 dbname char(32) The database name.
4 tabname char(32) The table name.
5 method char(16) The access method (see below).
6 keycolumn char(32) The key column used by the access method (or NULL)
7 stepcost bigint The optimizer's estimate of the number of logical disk accesses used in this step.
8 rows_in bigint The number of candidate rows from the prior step that are joined with the candidate rows in this step.
9 rows_out bigint The number of rows that result from this step.
10 plancost bigint The optimizer's estimate of the number of logical disk accesses for the entire plan. This value is the same in each row.

The access method used to retrieve rows at any particular plan step are described below.

Access Method Description
ROW COUNT Only used for "select count(*) from table"
VIRTUAL TABLE Access is to a virtual table either one row at a time (keycolumn=null) or through the specified keycolumn value.
TABLE SCAN Accesses each row of the table in the order stored in the file.
FK TO PK Accesses the primary key table row from the referencing row in the foreign key table which was accessed in a previous plan step. The keycolumn value will have identify the foreign key name.
PK TO FK Accesses the foreign key table row from the referenced row in the primary key table which was accessed in a previous plan step. The keycolumn value will have identify the foreign key name.
KEY FIND Accesses the rows through the keycolumn index for keys that match the specified keycolumn value.
KEY SCAN Accesses the rows by performing a scan of the keycolumn index.
KEY LIST Accesses the rows through the keycolumn index for keys that match the specified list of keycolumn values (e.g., the in relational operator).
KEY LIKE Accesses the rows through the keycolumn index for keys that match the specified keycolumn value pattern.
FK THRU PK KEY Accesses the rows of the foreign key table through its referenced primary key table that match the primary key value specified in keycolumn.

Example

The example below illustrates how rsqlShowPlan can be used to display the execution plan for any compiled SQL select statement.

#include "rsql.h"
 	
/* Display execution plan for specified statement handle
*/
RSQL_ERRCODE DisplayPlan(
   HSTMT       hQuery)
{
   RSQL_ERRCODE stat;
   HCONN        hdbc;
   HSTMT        hPlan;
   RSQL_VALUE  *row;
   uint16_t     nocols;
   char        *stmtstr;
   uint16_t     strlen;
   uint64_t     plancost;
   const char  *emsg;

   /* allocate handle for hQuery exec plan result set */
   stat = rsqlGetConnHandle(hQuery, &hdbc);
   if ( stat != errSUCCESS ) goto err_handler;

   stat = rsqlAllocStmt(hdbc, &hPlan);
   if ( stat != errSUCCESS ) goto err_handler;
    
   stat = rsqlShowPlan(hPlan, hQuery);
   if ( stat != errSUCCESS ) goto err_handler;

   /* get length of stmt string */
   rsqlGetStmtString(hQuery, NULL, 0, &strlen);  	

   /* allocate stmtstr buffer */
   stmtstr = psp_getMemory(++strlen, mtag);

   /* retrieve hQuery statement string */
   rsqlGetStmtString(hQuery, stmtstr, strlen, NULL);

   printf("\nEXECUTION PLAN FOR: \n%s\n\n", stmtstr);

   /* free stmtstr buffer */
   psp_freeMemory(stmtstr, mtag);

   /* display execution plan result set */
   printf("STEP TABLE                            METHOD         ");
   printf("KEYCOLUMN                        (COST, ROWS_IN, ROWS_OUT)\n");
   printf("---- -------------------------------- -------------- ");
   printf("-------------------------------- -------------------------\n");
	
   while ( rsqlFetch(hPlan, &row, &nocols) == errSUCCESS ) {
       /* ignore catalog and dbname (can be determined from stmtstr) */
       printf("%4d %-32.32s %-14.14s %-32.32s ", 
           row[0].vt.sv, row[3].vt.cv, row[4].vt.cv, 
           (row[5].type == tNULL ? "NULL" : row[5].vt.cv));
	    
       printf("(" INT64_SPEC ", " INT64_SPEC ", " INT64_SPEC ")\n", 
           row[6].vt.llv, row[7].vt.llv, row[8].vt.llv);
	        
       if ( row[0].vt.sv == 1 ) {
           /* save plan cost */
           plancost = row[9].vt.llv;
       }	    
   }
   printf("\nPLAN COST ESTIMATE: " INT64_SPEC "\n\n", plancost);
   rsqlFreeStmt(hPlan);
   return;

err_handler:
   rsqlGetErrorMsg(stat, &emsg);
   printf("**** %s\n", emsg);
   return;
}
