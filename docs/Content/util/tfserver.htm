tfserver

Transactional File Server

Prototype 

tfserver [-h] [-B] [-V] [-q|[-stdout file]] [-stderr file]  [-d PATH]
         [-p N] [-v] [-stdout filename] [-start|-stop|-query|
        -install exepath|-uninstall] [-rdonly] [-nodisk]

Description

The Transactional File Server (TFS) with default extensions is run by using the pre-built utility tfserver. All functions required to support the RDM runtime library are served by tfserver, as well as basic HTML files which can be stored in the httpdocs subdirectory under the document root (PATH). Some administrative functions are implemented into the XAPI of tfserver to support the browser-based administrative utility.

tfserver may run on the same computer as a runtime, or on a different computer, provided the different computer's domain name is visible to the runtime through TCP/IP. A runtime can locate the tfserver running on a different computer by using the environment variable RDM_TFSERVER. The value of RDM_TFSERVER is the domain name of the computer where tfserver is running, and optionally the port after a semicolon:

RDM_TFSERVER=RLM-lptp:1730

The default port is 21553, and if the -p option was not used when running tfserver, then the port field is not necessary in the RDM_TFSERVER value.

If tfserver is running on the same computer as the runtime, then the runtime may use only defaults, and no RDM_TFSERVER variable is needed. If the tfserver is using a non-default port (say, 3877), then the local runtime can use a RDM_TFSERVER setting as follows:

RDM_TFSERVER=localhost:3877

tfserver may be installed as a Windows service, or started as a daemon process. To determine if tfserver is already running, use the -query option (the -query option will work whether tfserver has been started in the foreground or background).

On Windows, you may install tfserver as a service that will run automatically by using the -install option. The PATH must list the fully qualified path to the tfserver.exe utility, or to the directory in which it resides. All required DLLs must reside either in the same directory as tfserver.exe, or in the Windows system32 directory. This option requires administrative privilege. To stop/start the tfserver service, use the Windows service control manager.

When this utility is run as a foreground process, it may be cleanly terminated by issuing SIGINT (normally ^C from the keyboard). SIGTERM has the same effect. If the utility does not terminate immediately it will be terminated immediately (cleanly or not) upon the third signal.

Options

-h Display this usage information
-B Do not display the banner
-V Display the version information
-q Quiet mode. No information will be displayed
-stdout file Redirect stdout to the specified file
-stderr file Redirect stderr to the specified file
-d PATH The path of the Document Root. Default is current directory.
-p N Anchor PORT, referring to the TFS associated with this utility. Default is 21553.
-v Verbose output.
-start Begin execution of utility as a separate process.
-stop Stop execution of utility as a separate process.
-query Check the status of the utility process (running/not running).
-install PATH Windows only, Requires administrative privilege. Install utility as an automatic service. PATH is the directory or fully qualified path to the utility's executable file.
-uninstall Windows only, Requires administrative privilege. Uninstall the utility service. Use the service control manager to stop the execution of the service first.
-rdonly Prohibit any operation that would modify the contents of the document root.
-nodisk Don't use any disk I/O.
