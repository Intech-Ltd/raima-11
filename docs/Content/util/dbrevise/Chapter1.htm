Introduction

1.1 Overview

The RDM database management system (DBMS) is designed to provide powerful, flexible, high-performance capabilities for developing C language database applications.  Raima's db_REVISE is a member of our database product family. A powerful companion program to the DBMS products, db_REVISE performs automatic revisions on any RDM database.

High performance in our products is achieved through the direct linkage between an application's code and the RDM database.  Because of our DBMS design, dynamic revision of the database structure is a complex process.  With db_REVISE, Raima database family products can be extensively altered with ease, without losing any existing data.  

New requirements, enhancements, optimizations, and bug fixes occur constantly in both the application software and in the database structure.  With db_REVISE you can create keys, insert or delete field types, add new record and set types, or join two record types into one.  Your task is to edit a copy of your schema (the DDL specification) and db_REVISE will do the rest, making the old data fit the new schema. 

The major features of the db_REVISE program include:

  • Adding record, set, and field types
  • Deleting record, set, and field types
  • Changing field types
  • Changing, adding, or deleting keys
  • Redistributing record types within files
  • Changing database page sizes
  • Projecting records into two or more component records
  • Reinitializing selected record and set types
  • Joining records through a path into a single record
  • Initializing field values
  • Splitting multiple-member set types
  • Combining multiple set types into multiple-member set types
  • Duplicating set connections
  • Normalizing sets during record projections

The db_REVISE program is written in C, with complete source code available.  It will operate within most of the same environments as other Raima database family products. In addition, db_REVISE comes in an object library form so that you can link it with your own field conversion function. Thus you are able to convert data fields in ways that are not pre-defined in db_REVISE.

db_REVISE and the db_REVISE User's Guide are intended for software engineers in the development of database application software.  As with the rest of Raima's family product members, db_REVISE was designed primarily for use in developing commercial applications and not as an end-user product.

db_REVISE must be used in conjunction with any 3.xx or later version of RDM.  Where RDM is used to design and manage a database, db_REVISE can be used to restructure that database.  You must be able to access the schema of the database you wish to revise. 

If the database was designed with a version of RDM earlier than 3.xx, you must run the ddlp corresponding to your version of RDM on a copy of the database schema file for use during the database revision process.  

1.2 RDM Documentation

RDM documentation is a three-manual set and includes this db_REVISE User's Guide, the RDM User's Guide and the RDM Reference Manual. 

The full set is provided online as HTML.   

1.2.1 About This Manual

This manual was designed to be self-instructional.  If this is your first encounter with db_REVISE, we recommend that you read all the sections.  Each section is a prerequisite to the next.

This manual documents the functions of the current version of db_REVISE.  Each section begins with an Introduction that gives a general idea of the section's subject matter. 

The manual is divided into sections as follows:

The Basics of Database Revision
Provides a quick overview of the usage of db_REVISE.  In a tutorial mode, it discusses the motivations for common database revisions and gives examples of how they can be performed. Also, the full set of command line options is explained.
The Three Classes of Revision
Provides the full functional specification of db_REVISE.  The three classes of revision (static, derived, and specified) are fully defined.
Examples of Common Operations
Provides examples of the most common uses of db_REVISE, using the tims database already familiar to users of RDM.
Errors and Warnings
Summarizes the error and warning messages db_REVISE prints, with some suggested causes and solutions.

1.2.2 Conventions

Table 1-1 describes the notational conventions used in this guide.

Table 1-1.  Document Notational Conventions
Convention Description

Bold Text

Indicates the names of functions, commands, data structures, statements, databases, utilities, files, etc.  Note that the case distinctions used in the names are not significant for all operating systems.  

Italic Text

Represents the names of variables, such as parameters and command options.  The RDM documents also use italics for the names of books.  

Courier Text

Indicates a programming example or a command line entry.  Programming examples in this book are enclosed in boxes.

Brackets ([ ])

Indicate optional fields or parameters in command lines and syntax statements.

Bar Symbol (|)

Separates alternative selections.  You should enter one of the items separated by the bar.

Ellipsis (...)

Indicates that you can repeat the preceding item.  Both horizontal and vertical ellipses are used in programming examples to indicate omitted code.

Arrow Symbols (< >)

Indicate information that you must supply.  For example, <filename>.h represents a header file that you name.  Arrow symbols are also used in text to set off keyboard key names, as in <Ctrl>.
