d_dbcheck

Database consistency check utility API.

Prototype

int32_t d_dbcheck(
    const DBCHECK_PARAMS *params,
    uint32_t             *num_errors,
    uint32_t             *num_warnings,
    DB_TASK              *task, 
    int32_t               dbn);

int32_t d_dbcheckW(
    const DBCHECK_PARAMSW *params,        /* UNICODE strings in structure */
    uint32_t              *num_errors,
    uint32_t              *num_warnings,
    DB_TASK               *task, 
    int32_t                dbn);

Parameters

params (input) A pointer to a DBCHECK_PARAMS structure.
num_errors (output) The number of errors detected during the dbcheck.
num_warnings (output) The number of warnings detected during the dbcheck.
task (input) A pointer to a DB_TASK structure.
dbn (input) The database number.

Description

This function invokes the dbcheck utility from your application. See the dbcheck utility page for a description of its operation.

DBCHECK_PARAMS structure
Member Type Description
quiet DB_BOOLEAN No output except for errors.
setscan uint16_t The -s option. If TRUE, scan sets for connection issues.
keyscan uint16_t The -k option. If TRUE, perform key file consistency check.
varscan uint16_t The -v option. If TRUE, perform vardata file structure consistency check.
blobscan uint16_t The -b option. If TRUE, perform BLOB file structure consistency check.
datkey uint16_t The -dk option. If TRUE, perform key access consistency check from data to key files.
keydat uint16_t The -kd option. If TRUE, perform key data consistency check from key to data files.
datvar uint16_t The -dv option. If TRUE, perform varchar field consistency check from data files.

vardat

 uint16_t The -vd option. If TRUE, perform data consistency check from varchar files.
datblob uint16_t The -db option. If TRUE, perform BLOB field consistency check from data files.
ignorekey uint16_t The -nk option. If TRUE, turn off keyscan, datkey and keydat and ignore key files.
ignorevar uint16_t The -nv option. If TRUE, turn off varscan, datvar and vardat and ignore vardata files.
ignoreblob uint16_t The -nb option. If TRUE, turn off blobscan and datblob and ignore blob files.
report uint16_t The -r # option. Report progress to stdout every # percent. Set to -1 for no report.
treetrace uint16_t The -t option. If TRUE, print a traceback of the b-tree at the first sign of disorder.
counts uint16_t The -c option. If TRUE, print counts of objects scanned in the check.
quick_dchain uint16_t The -qd option. If TRUE, only report one dchain error per file.
quick_keycheck uint16_t The -qk option. If TRUE, only report one error per key file.
log_errors uint16_t The -l option. If TRUE, log errors to a repair database.
fnames const char ** An array of file names listing the specific files to be checked. If this list is empty all files in the database will be selected.
nfnames uint16_t Number of files named in fnames.
stdout_file const char * This specifies the name of the file to which output data will be written. If set to PSP_STDOUT, stdout is used. If set to NULL or to an empty string, no output will be generated.

Currency Changes

Currency is invalidated by using this API.

Locking Requirements

This utility requires exclusive access to the database.

Example

/* RDM Database Consistency Check Utility API Example ******************************
*/
int32_t MAIN(int32_t argc, const char *const *argv)
{
    int32_t         status = S_OKAY;
    uint32_t        num_errors;
    uint32_t        num_warnings;
    DB_TASK        *task;
    DBCHECK_PARAMS  params;

    memset(&params, 0, sizeof(DBCHECK_PARAMS));

    /* just set the non-zero parameters */
    params.setscan  = 1;
    params.keyscan  = 1;
    params.varscan  = 1;
    params.blobscan = 1;
    params.datkey   = 1;
    params.keydat   = 1;
    params.datvar   = 1;
    params.vardat   = 1;
    params.datblob  = 1;
    params.stdout_file = PSP_STDOUT;

    
    if ((status = d_opentask(&task)) == S_OKAY) {
        if ((status = d_open("tims", "x", task)) == S_OKAY) {
            status = d_dbcheck(&params, &num_errors, &num_warnings, task, CURR_DB);
            d_close(task);
        }
        d_closetask(task);
    }

    return(0);
}
