dbexp

Database export utility.  The database export utility is capable of exporting data from the RDM database into either ASCII text files or a single XML file.  While this is one utility capable of working with either format, this section is broken into two parts for clarity, one dealing with the XML format, and the other dealing with the ASCII format.

Prototype

dbexp [-h] [-B] [-V] [-q|[-stdout file]] [-stderr file] [-tfs type]
      [-docroot path] [-d] [-e esc_char] [-encoding encoding] [-m]
      [-o filename] [-openmode mode] [-r] [-s sep_char] [-v] [-x]
      [-xml] [-btxt] [-xsd|-dtd] dbname [recname ...]

Options

-h Display this usage information
-B Do not display the banner
-V Display the version information
-q Quiet mode. No information will be displayed
-stdout file Redirect stdout to the specified file
-stderr file Redirect stderr to the specified file
-tfs TFS_TYPE Specify the TFS type to use (TFST, TFSR, or TFSS) [default = TFST]
-docroot path Document root for the TFS instance
-v Verbose - shows detailed output of the operations performed on the database.
-openmode mode Specify the open mode to use for the database (default 's' using ROT).

XML

Prototype

dbexp [-encoding encoding] [-o filename] [-xml] [-btxt] [-xsd|-dtd] dbname

Description

This utility exports data from database dbname to an XML file (dbname.xml).  By exporting data into XML format, consumers of the data can be RDM (e.g. for database restructuring or archiving) or other XML consumers.  The XML content created by the dbexp utility will allow reconstruction of sets (in the case of RDM as the consumer), or interpretation by any normal XML consumer.

Options

-xml Specifies output in XML format (XML mode)
-xsd Creates XML Schema (dbname.xsd)
-dtd Creates DTD (dbname.dtd)
-o filename Send output to filename
-encoding enc Supported encodings are UTF-8 and ISO-8859-1 (Default is UTF-8)

ASCII

Prototype

dbexp [-d] [-e esc_char] [-m] [-r] [-s sep_char] [-v] [-x] [-xml] [-btxt]
      dbname [recname ...]

Description

This utility exports data from database dbname to ASCII text files that can then be moved to another computer, or used as input into other tools that read ASCII formatted files (for example, dbimp). The dbexp utility writes all data from each record type to a separate file; it is the record name with file extension .txt. If a record name, recname, is specified on the command line, then only that record type's occurrences are exported. The created files will consist of one text line per record occurrence with commas separating the data fields.

The dbexp utility has two ways of printing extended ASCII characters to the output files. The default is to write out the octal value. Since dbimp cannot handle this method, the -x option can be used to print the extended ASCII character as the character. For example, ASCII 2 would be printed as "\002" without the -x and as a smiley face with the -x option.

Options

-d Causes a database address to be printed as long integers, rather than in the standard format.
-e ch Changes the escape from a backslash ('\') to character ch. White space between the "e" and the character is optional.
-m Prints the database addresses of the owners of all sets for which the record is a member.
-r Prints a record's database address as the first field of each record occurrence.
-s ch Changes the field separator character from a comma to character ch. White space between the "s" and the character is optional.
-x Prints extended ASCII characters in octal.
-btxt Export blob fields as ASCII text. Non-SQL databases will treat blob fields as binary data and export the contents of the field as an array of hex characters.
