rdmsql

Simple Interactive SQL Utility

Command Syntax

rdmsql [-h] [-B] [-V] [-q|[-stdout file]] [-stderr file] [-tfs type]
       [-docroot path] [-d path] [-b] [-c] [-a tfs] [-e] [-u] [-r]

Description

The rdmsql utility is a simple interactive SQL utility based which uses the ODBC API. This allows the utility to use the default TFS mode for connecting with the database as well as provides the ability to connect to a remote rdmsqlserver utility running on another system.

Options

-h Display this usage information
-B Do not display the banner
-V Display the version information
-q Quiet mode. No information will be displayed
-stdout file Redirect stdout to the specified file
-stderr file Redirect stderr to the specified file
-tfs TFS_TYPE Specify the TFS type to use (TFST, TFSR, or TFSS) [default = TFST]
-d[ocroot] path Document root for the TFS instance
-b Run startupfile in batch mode
-c Run startupfile in compilation mode
-a host[:port] Name of tfserver to use,
-e Disable echo of startupfile commands.
-u Display result set headings in uppercase.
-r Use remote connection to rdmsqlserver
-C # Set maximum database connections to # (default 5)
-H # Set size of statement history list to #
-s # Set maximum statements per connection to # (default 5)
-w # Set maximum display page width to # (default 4096).
-o errorfile Specify error log

Comments

startupfile [args]

Text file containing scripted commands for rdmsqlserver to execute.

-d[ocroot] path

The docroot option is used only when rdmsqlserver uses a direct-link (TFST) or standalone (TFSS).

-a hostname[:port]

The hostname option is used only when rdmsqlserver uses an external tfserver (TFSR).

-c
-b

The compile (-c) and batch (-b) options automatically connect to the tfserver (local for TFST/TFSS or the "-a" option above for TFSR), perform the scripted commands, commit the changes and then exit. These modes are typically used for compiling the database schema (DDL).  The "-c" option enables an additional option, "Generate C Files", used for embedding the initialized database into the application program.

-o errorfile

Errors encountered during the execution of the startupfile are output to stdout. This option allows those messages to be redirected to a file.
