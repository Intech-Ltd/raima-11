rdmsqlserver

Remote database connectivity server

Command Syntax

rdmsqlserver [-h] [-B] [-V] [-q|[-stdout file]] [-stderr file] [-tfs type]
             [-docroot path] [-d path] [-p N] [-v] [-stdout filename]
             [-start|-stop|-query|-install exepath|-uninstall]

Description

rdmsqlserver is a command-line utility that lets an RDM ODBC/JDBC application manage RDM SQL databases stored on a remote computer. The RDM ODBC/JDBC libraries communicate with rdmsqlserver through TCP/IP. rdmsqlserver uses the RDM SQL library to access a tfserver that manages the RDM SQL databases.

rdmsqlserver can reside on the same computer as the RDM ODBC/JDBC application does, or on a remote computer. In the case of the latter, the architectures of the computer where rdmsqlserver is running does not need to match that of the computer that runs the RDM ODBC/JDBC application. Note that rdmsqlserver and the target tfserver must be on comparable platform architectures.

rdmsqlserver can be executed as a foreground process or as a daemon process. It can also be installed as a service on Windows. To determine if rdmsqlserver is already running, use the -query option (the -query option will work whether rdmsqlserver has been started in the foreground or background).

When this utility is run as a foreground process, it may be cleanly terminated by issuing SIGINT (normally ^C from the keyboard). SIGTERM has the same effect. If the utility does not terminate immediately it will be terminated immediately (cleanly or not) upon the third signal.

Options

-h Display this usage information
-B Do not display the banner
-V Display the version information
-q Quiet mode. No information will be displayed
-stdout file Redirect stdout to the specified file
-stderr file Redirect stderr to the specified file
-tfs TFS_TYPE Specify the TFS type to use (TFST, TFSR, or TFSS) [default = TFST]
-d PATH The path of the Document Root. Default is current directory.
-p N Anchor PORT, referring to the TFS associated with this utility. Default is 21553.
-v Verbose output.
-start Begin execution of utility as a separate process.
-stop Stop execution of utility as a separate process.
-query Check the status of the utility process (running/not running).
-install PATH Windows only, Requires administrative privilege. Install utility as an automatic service. PATH is the directory or fully qualified path to the utility's executable file.
-uninstall Windows only, Requires administrative privilege. Uninstall the utility service. Use the service control manager to stop the execution of the service first.
-nodisk Don't use any disk I/O.
-notfs Do not allow direct connection to TFS

Comments

-d[ocroot] path

The docroot option is used only when rdmsqlserver uses a direct-link (TFST) or standalone (TFSS).

-p N

The port number option specified for rdmsqlserver is called an "anchor port," or the base port number off which we generate an appropriate port number for our other utilities that use port numbers.

For external tfserver (TFSR) configurations the port number must be the port number for the external tfserver running on the same computer.

For embedded and external tfservers (TFST and TFSR), rdmsqlserver will use the port number provided + two (2) to calculate the port number to be used as the listening port for RDM ODBC/JDBC connections. For example, if port 5000 is specified with "-p" (i.e. "-p 5000"), rdmsqlserver will listen on port 5002.

-install exepath

On Windows, you may install rdmsqlserver as a service that will run automatically by using this option. The exepath must be the fully qualified path to the rdmsqlserver executable, or to the directory in which it resides. All required DLLs must reside either in the same directory as rdmsqlserver, or in the Windows system directory. This option requires administrative privilege. To stop/start the rdmsqlserver service, use the Windows service control manager.
