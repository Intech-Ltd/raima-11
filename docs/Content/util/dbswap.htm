dbswap

Database byte-swap utility

Prototype 

dbswap [-docroot path] [-b] [-D] [-o] [-v] [-q] dbname   ]

Description

The dbswap utility performs byte-swap operations on a database allowing it to be used on a platform utilizing an integer endian format different than the platform the database was created on.

This utility operates directly on the database, not through the TFS. This means that the utility must be run on the same computer as the database, and must specify a correct path to the database - either a full path or a path relative to the current directory. The database is found in the TFS document root directory in a subdirectory of the same name as the database. For example, if the document root is c:\RDM\databases, then the following sequence would perform the swapping operation in the database named sales:

cd c:\RDM\databases\sales
dbswap sales

RDM database images are created in a binary format specific to the platform it is generated on. This means that a database image created on a big-endian platform is incompatible with an RDM runtime running on a little-endian platform. This utility will convert the endianness of a database image allowing it to be read on platforms other than the one it was generated on. This is particularly useful for hosted development environments where the host and target use processor architectures of differing endian format.

Dbswap is not compatible with Unicode fields. RDM utilizes different encoding formats for Unicode characters on different platforms. This version of dbswap does not implement a transcoding scheme that would allow those fields to be transferred across differing architectures.

The suggested procedure for converting a database image for databases that contain Unicode data fields is to use the dbexport utility to create an XML image on the host. You can then use the dbimp utility to populate the database on the target from the XML image.

Procedures

  1. Run dbcheck on your database and ensure that the database is valid.
  2. Backup your database.
  3. Run the dbswap utility.

Options

-q Quiet - suppresses all output (except errors)
-docroot path [non-RPC configuration only] Specifies the document root when in Direct-Link or Standalone configuration.
-v Verbose - shows detailed output of the operations performed on the database.
-b The dbd file is not native to the platform dbswap is running on.
-D Swap only the dbd file, do not swap the database contents.
-o Overwrite the existing database files.
dbname Specifies the database to swap.
