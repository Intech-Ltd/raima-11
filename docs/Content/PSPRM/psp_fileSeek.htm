psp_fileSeek

Seek to a byte offset position in a file

Prototype 

uint32_t psp_fileSeek(
    PSP_FH          handle,
    PSP_FILE_OFFSET addr,
    int32_t         origin);

Parameters

handle (input) The handle to the PSP file opened with psp_fileOpen
addr (input) The position/offset within the file relative to the origin.
origin (input) The point from which to position addr bytes.

Description

This function sets the position of a file to the specified address. The address is the byte offset from the origin. Origin values follow standard I/O conventions:

PSP_SEEK_SET position to beginning of file plus addr
PSP_SEEK_CUR position to current position plus addr. New current position results.
PSP_SEEK_END position to end of file minus addr.

This function should not be used with files shared between multiple threads. The conflict occurs because, in the time between the call to psp_fileSeek and the subsequent call to psp_fileRead or psp_fileWrite, another thread could have issued a call to psp_fileSeek, changing the file position. Use psp_fileSeekRead or psp_fileSeekWrite for shared, multi-threaded file access.

Required Headers

#include "psp.h"

Libraries

Library Name Description
rdmpsp11 Platform Support Library

See Library Naming Conventions section for full library name and a list of library dependencies.

Return Codes

Value Name Description
0 PSP_OKAY A normal return; successful.
2 PSP_FAILED  

See Also

psp_fileSeekWrite, psp_fileSeekRead, psp_fileRead, psp_fileWrite, psp_fileOpen

Example 

#include "psp.h"
    
    int16 readFileAt(PSP_FH hFile, PSP_FILE_OFFSET addr, void *buf, uint32_t tSize)
    {
        uint32 len;
        
        psp_fileSeek(hFile, addr, PSP_SEEK_SET);
        len = psp_fileRead(hFile, buf, tSize);
        if (len != tSize)
            return PSP_FAILED;
        
        return PSP_OKAY;
    }
