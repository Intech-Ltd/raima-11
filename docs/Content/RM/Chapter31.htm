Introduction

This section contains descriptions of the application program interface (API) of all RDM runtime library functions. Each page header displays the function name described on that page. The Prototypes section gives a synopsis of the actual function declaration (not how it is called). The Parameters section describes all parameters.

Note: The parameters for file, record, field, and set identifiers appear in uppercase. This indicates that normally the corresponding C constant created by ddlp in <dbname>.h is passed to the function.

The Description explains all of the function's operational details. Any Currency Changes are listed using assignment statements in the order in which the changes occur. The notation "curr_own[SET]," is read "current owner of SET". In the event of a user or system error, the changes to the currency table are not made.

When the multi-user library runs in shared mode, many functions have Locking Requirements. The lowest level is no locking, or "None." The next level is "None, but may be old or deleted data," (see Read Locks section in the RDM User's Guide). The third level is "Read lock required." The highest level is "Write lock required." These requirements may be ignored if the database is opened in exclusive or mode.

The User's Guide section points to explanatory material in the User's Guide, saving you search time when you need a full description of the concepts involved.

In the Return Codes section, the most common status and error codes are listed, often with a possible explanation for their occurrences. We do not list all possible codes, but only those that tend to occur most often.

Related functions are identified in the See Also section.

An Example section is provided for most functions. The example is intended to illustrate how the function is called, not necessarily how to best use the function. In some cases, the example is brief, but it should be sufficient to show how the call is made. Database Manipulation, Transaction Processing, and Multi-User Database Control, of the RDM User's Guide provide many more examples of RDM functions.
