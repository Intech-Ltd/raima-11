Summary Listing of XML Import/Export Functions

These functions require the header file "rdm.h" included in source files. To link these functions, the library name 'xml' must be included (see Library Naming Conventions for library naming conventions).

Summary of XML Export functions

The following table lists alphabetically all the XML Import/Export functions, along with brief descriptions.

Table 3-4. Listing of XML Import/Export functions
Function Description
exp_init Initiate export
exp_term Terminate export
exp_xmlschema Export a database dictionary to XML Schema or DTD format
exp_xmlschemarec Export a record definition to XML Schema or DTD format
exp_xmlrec Export a record from the database to XML format
exp_xmlgetownertypes Get a list of record types that will be read from the database when the specified record is exported

Summary of XML Import functions

The following table lists alphabetically all the XML Import functions, along with brief descriptions.

Function Description
imp_init Initiate import
imp_term Resolve forward references, remove temporary databases and free memory associated parser and user data environment
imp_xmlParse Parse more of the document.
imp_xmlGetErrors Get details of most recent XML conversion error
imp_xmlSetRecordCallback Set customer record processing callback.
imp_xmlSetErrorCallback Set database error callback.
imp_xmlSetConversionCallback Set custom data conversion callback.

Callback functions

User defined callback functions can replace or enhance the default behavior of the import function. The return code from the callback function signals the import to continue with the default processing or signal that processing is complete. The user defined callback functions must return one of the following return codes.

Return Codes Description
IMPXML_CONTINUE Continue with default processing.
IMPXML_BREAK Skip default processing.
