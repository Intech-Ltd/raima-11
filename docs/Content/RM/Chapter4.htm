Return Codes

This section describes the status and error codes returned by the RDM runtime functions. The descriptions in this section explain the conditions under which that code would be returned, and, where possible, guidelines are given for resolving the problem.

For more information on RDM error handling, see the Database Error Reporting section in the RDM User's Guide.

Note that if you are testing the bounds of error messages within your dberr routine (see also d_set_dberr), you can use the following #defines:

S_USER_FIRST and S_USER_LAST

S_SYSTEM_FIRST and S_SYSTEM_LAST

S_FUNCTION_FIRST and S_FUNCTION_LAST

For example, to test for a user error, use the following:

if ((errnum <= S_USER_FIRST) && (errnum >= S_USER_LAST))
