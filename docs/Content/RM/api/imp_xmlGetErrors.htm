imp_xmlGetErrors

Get details of most recent XML conversion error

Prototype

void imp_xmlGetErrors(
    DB_IMPENV   *env, 
    uint32_t    *nerrors, 
    DB_IMPERR **errors);

Parameters

env (input) Import session data
nerrors (output) number of array elements returned in errors – the caller may pass NULL if this result is not required
errors (output) array of DB_IMPERR structures containing details of errors – the caller may pass NULL if this result is not required

Description

This function may be called after the function imp_xmlParse when an error condition is returned. The list of errors returned by imp_xmlGetErrors is in the form of an array of DB_IMPERR structures, as defined below. The number of errors will be returned in nerrors, which will be zero if there were no errors encountered.

DB_IMPERR structure definition
Element Declaration Description
line uint32_t line number of the error. The line number begins counting from the beginning of the first call to imp_xmlParse.
code int32_t error code (See Import Error Codes below)
text const char * error text corresponding to error code.
name const char * name of field, record or set found in XML.

The pointer returned in errors points to memory allocated internally by the import functions, which will be freed when imp_term is called. The contents of this memory may be overwritten by the next call to imp_xmlGetErrors, so the calling application should copy the contents if it needs to preserve them.
