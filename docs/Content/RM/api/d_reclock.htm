d_reclock

Lock files associated with record type

Prototype

int32_t d_reclock(
    int32_t     REC, 
    const char *type, 
    DB_TASK    *task, 
    int32_t     dbn);
int32_t d_reclockW(
    int32_t        REC, 
    const wchar_t *type,  /* UNICODE string */
    DB_TASK       *task, 
    int32_t        dbn);

Parameters

REC (input) The record type for which the files are to be locked.
type (input) A pointer to a string containing the type of lock to be applied.
task (input) A pointer to a DB_TASK structure.
dbn (input) The database number.

User's Guide

File Locking

Upgrading/Downgrading of File Locks

Description

This function places a lock on the data and key files associated with REC. The lock type is specified as follows.

  • "r" (read lock)
  • "w" (write lock)
  • "k" (keep lock)

Read locks allow other programs to read but not update the locked files. Write locks prevent all other programs from accessing the locked files and can only be issued from within a transaction. Keep locks can only be issued from within a transaction and will keep read and write-locked files read-locked after a transaction ends (d_trend normally frees all read and write-locked files). The record type must be locked in order to apply a keep lock; otherwise, status S_NOTLOCKED is returned.

Within a transaction, a read-locked file can be upgraded to a write or exclusive lock by calling d_reclock with a write lock where a read lock already exists. If the upgrade lock request is denied, the read lock will remain in effect. However, the denied upgrade request means that another task has made the upgrade request first, and will time out if the read lock is not freed. Thus, upgrade denial should be followed by re-starting the transaction or by freeing all read locks.

Currency Changes

None.

Locking Requirements

None.

Return Codes

Value Name Description
0 S_OKAY normal return, okay
5 S_UNAVAIL database file currently unavailable
-3 S_INVREC invalid record
-11 S_BADTYPE invalid lock value
-23 S_NOTFREE illegal attempt to lock locked set/record
-31 S_NOTRANS cannot update database outside a transaction
-33 S_STATIC locks not allowed on static files
-79 S_INROT Cannot apply a write or exclusive lock while in a read-only transaction
-939 S_READONLY unable to update file due to READONLY option

Example

int32_t count, stat = S_OKAY;
   ...

/* list info record in id_code order */
while ( stat == S_OKAY )
{
   while ( d_reclock(INFO, "r", task, CURR_DB) == S_UNAVAIL )
      	;
   for ( count = 0; 
       (count < 20) && ((stat = d_keynext(ID_CODE, task, CURR_DB)) == S_OKAY);
       ++count)
   {
       /* hold lock for only 20 records at a time */
       d_recread(&irec, task, CURR_DB);
       pr_info();
   }
   d_recfree(INFO, task, CURR_DB);
}
