d_trmark

Mark a transaction

Prototype

int32_t d_trmark(
    const char *mtag,
    DB_TASK    *task);

Parameters

mtag (input) A pointer to the current position in the transaction.
task (input) A pointer to a DB_TASK structure.

Description

This function marks the specified position within the current transaction.  Later, your application can roll back (via d_trrollback) to this point in the transaction.  The application uses d_trmark and d_trrollback to discard portions of a transaction without terminating the transaction or discarding locks.

Currency Changes

None

Locking Requirements

None

Example

/* Add two records of type sRec
   with data pointed to by pData1 and pData2
   into the first databases opened using task.
   A transaction must be active, record type sRec
   must be locked for writing, and the data must be
   unique with respect to any unique key fields.
   This function is transactional and will return
   with an error if the precondition is not satisfied.
*/

int32_t addTwoRecords (
    int16_t  sRec,
    void    *pData1,
    void    *pData2,
    DB_TASK *task)
{
    int32_t stat= d_trmark ("mark_addTwoRecords", task);

    if (stat == S_OKAY) {
        stat= d_fillnew (sRec, pData1, task, 0);
    }

    if (stat == S_OKAY) {
        stat= d_fillnew (sRec, pData2, task, 0);
        if (stat != S_OKAY) {
            d_trrollback ("mark_addTwoRecords", task);
        }
    }
    
    return stat;
}
