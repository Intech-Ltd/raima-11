d_dbsetini

Copies a value into the specified section of a database initialization file.

Prototype

int32_t d_dbsetini(
    const char *section,
    const char *key,
    const char *value,
    DB_TASK    *task, 
    int32_t     dbn);

Parameters

section (input) The name of the section containing the key name. If the section does not exist, it is created.
key (input) The name of the key to be associated with a value. If the key does not exist in the specified section, it is created.
value (input) A null terminated value to be written to the file.
task (input) A pointer to a DB_TASK structure.
dbn (input) The database number.

Description

This function sets configuration settings for the current database dbn. If a database initialization file does not exist, the file will be created.

The database must not be a union database.

The replication key value in the configuration section should not be changed if other threads or processes have the database already open.

If replication is enabled, the database cannot be opened in exclusive mode.

A section in the initialization file will have the following form:

[section]
key=value
    .
    .
    .

TFS Configuration Settings

None.

Database must be opened in exclusive mode.

Return Codes

Value Name Description
0 S_OKAY normal return, okay
-4 S_INVDB cannot open dictionary
-32 S_EXCLUSIVE exclusive access required
-51 S_INVFORUNIONDB operation invalid for a union database

Example

int32_t status;

status = d_open("tims", "x", task);
if ( status != S_OKAY ) {
    if ( status == S_UNAVAIL )
        printf("database(s) not currently available\n");
    return( status );
}
/* Turn on mirroring capability */
d_dbsetini("configuration", "mirroring", "1", task, CURR_DB);
