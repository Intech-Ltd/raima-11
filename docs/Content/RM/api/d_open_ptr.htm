d_open_ptr

Open database using internal dbd array

Prototype

int32_t d_open_ptr(
    const char *dbname, 
    const char *type, 
    const void *dbd_array, 
    int32_t     dbdsize, 
    DB_TASK    *task 
);
int32_t d_open_ptrW(
    const wchar_t *dbname,    /* UNICODE string */
    const wchar_t *type,      /* UNICODE string */
    const void    *dbd_array, 
    int32_t        dbdsize, 
    DB_TASK       *task 
);

Parameters

dbname (input) Name of the database to be opened.
type (input) A string containing the type of access to the database.
dbd_array (input) dbd array generated by the ddlp -c option.
dbdsize (input) sizeof(dbd_array).
task (input) A pointer to a DB_TASK structure.

Description

This function is the same as the d_open function except that the database dictionary (DBD file) is supplied in the parameter list (as dbd_array and dbdsize) rather than found in a dbname.dbd file on the TFS. This is useful when a program may need to initialize a new database that isn't already present on a TFS. It may also be used when the TFS is using memory to store database files, because there is no dependency on the existence of the DBD file on the TFS.

If a .dbd file does not exist in the database directory on the TFS, this function will create it. If the database is defined as inmemory, the .dbd file will be placed into the in-memory storage together with the other database files. Regardless of whether the .dbd is stored in memory or on disk, this allows an application to create a .dbd on the TFS that is then available to utilities or other programs that have not been compiled with the database's dictionary embedded into them.

However, if a .dbd file does exist in the database directory on the TFS, this function will not overwrite it. Care should therefor be taken if you modify the schema. Make sure no .dbd file is present on the TFS. Deleting the database by doing d_destroy will delete the .dbd and all the database content.

To use this function, it is necessary to create dbd_array. This is done by running ddlp with the -c option, which generates two additional files. The first, dbname_dbd.c, must be compiled and linked with the program. The second, dbname_dbd.h, must be included in the program source. Also in the program source, the d_open_ptr call is supplied with the dbd_array and dbdsize parameters as "&dbname_dbd, sizeof(dbname_dbd),".

Since only one dictionary is represented by dbd_array, it is not possible to open multiple databases, as it is in d_open.

Refer to d_open for more information on opening databases.
