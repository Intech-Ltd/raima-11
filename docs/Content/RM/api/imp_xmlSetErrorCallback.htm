imp_xmlSetErrorCallback

Set database error callback.

Prototype

void imp_xmlSetErrorCallback(
    DB_IMPENV    *env, 
    IMPXMLERRPROC errfunc);

Parameters

env (input) Import session data
errfunc (input) User defined custom error processing function. The valid return values for the errfunc are:
IMPXML_CONTINUE - Continue with default processing
IMPXML_BREAK -Skip default processing

User defined callback functions can replace or enhance the default behavior of the import function. The return code from the callback function signals the import to continue with the default processing or signal that processing is complete. The user defined callback functions must return one of the above return codes.

Description

The error callback function will be called during the default record processing for any error condition returned by any database function call. The default behavior for a database function error code (errorcode > 0) will be to skip the processing of the current record and continue. System and User error codes (errorcode < 0) will halt the XML parsing.

IMPXMLERRPROC typedef

typedef int32_t (*IMPXMLERRPROC)(DB_IMPENV *env,
        int32_t errorcode);
