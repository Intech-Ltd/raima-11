d_trend

End transaction

Prototype

int32_t d_trend(
    DB_TASK *task);

Parameters

task (input) A pointer to a DB_TASK structure.

Description

This function ends a database transaction by writing to the database log file all changes made since the most recent call to d_trbegin. The changes are first written to the user log file to support recovery and are then written to the database. The process of writing changes to the database is called a transaction.

Except for keep-locked files, d_trend frees all locked files for all databases in a task opened by an application program.

Currency Changes

None.

Locking Requirements

None.

Return Codes

Value Name Description
0 S_OKAY normal return, okay
-27 S_TRNOTACT transaction not active
-29 S_TRFREE cannot free locks within a transaction
-222 S_TX_ILLEGAL_MOD Illegal attempt to modify file(s) on read-only device
