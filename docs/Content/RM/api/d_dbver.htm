d_dbver

Return runtime library version information

Prototype

int32_t d_dbver(
    const char *fmt, 
    char       *buf, 
    int32_t     len);
int32_t d_dbverW(
    const wchar_t *fmt, 
    wchar_t       *buf, 
    int32_t        len);

Parameters

fmt (input) A string that specifies how the version information is to be formatted.
buf (output) A string buffer to receive the formatted version string.
len (input) Length of *buf in characters.

Description

This function fills a buffer with a string describing the version of the RDM library or DLL according to a format string. The format string consists of plain text (copied to the string buffer) and format specifiers introduced by a percent ("%") character. See list below.

d_dbver Type Field Characters
Character Output Description
%n Raima Database Manager Product Name
%v 11.0.0 Version number
%b 412 Build number
%V Raima Database Manager11.0.0 Build 412 [2-15-2012] Full version string
%c Copyright (c) 2012 Copyright date
%w Raima Inc. Owner name
%r All Rights Reserved. Rights text
%C Copyright (c) 2012 Raima Inc., All Rights Reserved. Full Copyright string
%% % Literal percent character

User's Guide

Database Control

Required Headers

#include "rdm.h"

Libraries

Library Name Description
rdmrdm11 RDM Runtime Library

See Library Naming Conventions section for full library name and a list of library dependencies.

Return Codes

Value Name Description
0 S_OKAY normal return, okay

Example

#include <stdio.h>
#include <rdm.h>

void credits(void)
{
    char buf[80];
    printf("----- Credits -----\n\n");
    printf("This application is RDM Based \n");
    d_dbver("   %n\n   Version %v %b\n", buf, 80);
    printf(buf);
    d_dbver("   %C\n", buf, 80);
    printf(buf);
}
