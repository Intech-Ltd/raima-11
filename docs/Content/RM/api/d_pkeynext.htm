d_pkeynext

Find the record occurrence associated with the next key

Prototype

int32_t d_pkeynext(
    int32_t     keyfield,
    uint16_t    nFields,
    uint16_t    partialStrLen,
    const void *fldval,
    DB_TASK    *task,
    int32_t     dbn);

Parameters

keyfield (input) The field type of the key field for which the next key will be found.
nFields (input) The number of fields to use in the search.  If this parameter is zero, the function searches all fields.  If this parameter is a non-zero value, the function uses only the number of fields indicated.  This parameter is intended only for compound keys, although specifying '1' for non-compound keys is safe.
partialStrLen (input) The partial string length in characters.  If this parameter is a non-zero value, the function uses only this many characters of the last field being considered (depending on nFields above).
fldval (input) A pointer to the data to be searched.
task (input) A pointer to a DB_TASK structure.
dbn (input) The database number.

Description

This function finds the record occurrence associated with the next occurrence of the specified key (keyfield parameter).  The d_pkeynext function with nFields equal to zero and partialStrLen equal to zero is equivalent to d_keynext.

The d_pkeynext function returns S_NOTFOUND if it finds no more keys or a key that is greater than the key specified by fldval (as restricted by nFields and partialStrLen).  Because RDM keeps track of the relative position of each key field in a key stack, the application can intersperse d_pkeynext and d_keynext calls for multiple key fields.

Note that the data provided to d_pkeynext does not have to match the data provided to the corresponding d_pkeyfind. This allows range lookups. For instance, to find all the keys in the range 100-199, you could call d_pkeyfind with the value 100. If 100 existed in the database it would return S_OKAY, otherwise it would return S_NOTFOUND. In either case you could then call d_pkeynext with the value 199 and it would return S_OKAY until all the values up to, and including, 199 were accessed.

#include "rdm.h"

Return Codes

Value Name Description
0 S_OKAY normal return, okay
2 S_NOTFOUND record not found
-17 S_NOTKEY field is not a valid key
-24 S_NOTLOCKED attempt to access unlocked set/record
-44 S_INVPKEY Invalid partial key specification
-47 S_INVINDEX Invalid index type

Currency Changes

curr_rec = record with next key; (or NULL_DBA if S_NOTFOUND)

Locking Requirements

None.  However, if the record is not locked, your application might receive an S_DELETED error code when it later attempts to read the record found by d_pkeynext.  This code indicates that another application has deleted the record.  Also, another application might change the record so that its key field no longer matches the value specified to d_pkeynext.
