d_trrollback

Roll a transaction back to a mark

Prototype

int32_t d_trrollback(
    const char *mtag,
    DB_TASK    *task)

Parameters

mtag (input) A pointer to the marker within the transaction to which this function rolls back.
task (input) A pointer to a DB_TASK structure.

Description

This function rolls back within a transaction to the position indicated by mtag (previously set via d_trmark or d_trbegin).  The comparison of mtag with mark tags is case-sensitive.  If d_trrollback cannot find the marker (mtag), it rolls back to the beginning of the transaction, which remains active.  The d_trrollback function does not free locks on existing records and sets obtained in a transaction. 

Your application can use d_trbegin, d_trmark, and d_trrollback to discard portions or all of a transaction.

Currency Changes

The state of the currency can be undefined following completion of the d_trrollback function.  The application should always explicitly re-establish the currency.

Locking Requirements

None
