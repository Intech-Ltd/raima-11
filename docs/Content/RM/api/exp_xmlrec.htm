exp_xmlrec

Export a record from the database to XML format

Prototype

int32_t exp_xmlrec(
    size_t    *nbytes, 
    char     **xmlbuf, 
    DB_ADDR    addr, 
    DB_EXPENV *env);

Parameters

nbytes (output) number of bytes, including null terminator, returned in xmlbuf – the caller may pass NULL if this result is not required
xmlbuf (output) buffer containing XML representation of record being exported – the caller may pass NULL if this result is not required
addr (input) database address of record to export
env (input) export state data

Description

This function reads the record at the specified database address and converts it to XML. If there are no unique key fields in the record, a rowid value derived from its database address is included in the XML output. If the record is a member of sets then appropriate relational references to the owners are included in the XML output. Since a relational reference may consist of a reference to a unique key value in the owner record, this function may have to read some owner records. The calling application must ensure that these owner records are read-locked, unless dirty reads are acceptable. The function exp_xmlgetownertypes can be used to determine which record types must be locked prior to calling this function.

The XML output is generated in the encoding format specified (default is UTF-8). The value returned in nbytes always specifies a number of bytes.

Before generating any XML, this function allocates a large internal buffer for the XML output, unless one has already been allocated during a prior call to an export function. As the XML record is generated and text is appended to the output, the function may reallocate and extend the internal buffer as required.

The address returned in xmlbuf points to this internal buffer, which is allocated by the export functions and freed by exp_term. The contents of the buffer may be overwritten by the next call to exp_xmlrec, exp_xmlschema or exp_xmlschemarec, so the calling application should copy the contents if it needs to preserve them.

Locking Requirements

No locks required, but, to avoid dirty reads, read locks should be placed on the record type being exported and all owner record types, as returned by exp_xmlgetownertypes.

Return Codes

Value Name Description
0 S_OKAY normal return, okay
6 S_DELETED record/set deleted since last accessed
-1 S_DBOPEN database not opened
-904 S_NOMEMORY out of memory

Example

    #include "tims.h"

    DB_TASK     *task;
    DB_EXPENV   *env;
    int32_t      numtypes;
    int32_t     *types;
    int32_t      ii;
    int32_t      stat;
    DB_ADDR      dba;
    char        *xmlbuf;
    size_t       bytes;

    d_opentask(&task);
    d_open("tims", "s", task);

    exp_init(&env, "UTF-8", task, CURR_DB);

    /* Output the description of the INTERSECT record */    
    exp_xmlschemarec(&bytes, &xmlbuf, DB_EXP_XML_SCHEMA, INTERSECT, env);
    fprintf(stdout, "%s\n", xmlbuf);
    
    exp_xmlgetownertypes(&numtypes, &types, INTERSECT, NULL_DBA, env);
    for (ii = 0; ii < numtypes; ++ii)
    {
        /* Must get read locks on owner records */
        if ((stat = d_reclock(types[ii], "r", task, CURR_DB)) != S_OKAY)
            return stat;
    }
    
    /* Output all INTERSECT records */
    for (stat = d_recfrst(INTERSECT, task, CURR_DB);
         stat == S_OKAY;
         stat = d_recnext(task, CURR_DB))
    {
        d_crget(&dba, task, CURR_DB);
        if (exp_xmlrec(&bytes, &xmlbuf, dba, env) == S_OKAY)
            fprintf(stdout, "%s\n", xmlbuf);
    }
    
    d_freeall(task);
    exp_term(env);
    
    d_close(task);
    d_closetask(task);
