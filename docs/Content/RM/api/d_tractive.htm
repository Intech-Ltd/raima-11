d_tractive

Determine whether a transaction is active

Prototype

int32_t d_tractive(
    int16_t    *bool,
    DB_TASK    *task);

Parameters

bool (output) A pointer to the transaction state flag. The flag is TRUE if the transaction is active and FALSE if the transaction is not active.
task (input) A pointer to a DB_TASK structure.

Description

This function determines if a database transaction is active in the session specified by task. Since RDM does not allow nested transactions, the application might call d_tractive within subfunctions to determine whether to begin and end a transaction. If a transaction is already active, the application can leave calls to d_trbegin and d_trend to higher levels in the code.
