d_iopen

Open database incrementally

Prototype

int32_t d_iopen(
    const char *dbnames, 
    DB_TASK    *task);
int32_t d_iopenW(
    const wchar_t *dbnames,   /* UNICODE string */
    DB_TASK       *task);

Parameters

dbnames (input) Name(s) of the databases to be opened.
task (input) A pointer to a DB_TASK structure.

Description

This function incrementally opens additional databases, and can only be called after a call to d_open (only d_open can establish the type of database access). Databases opened by d_iopen will use the same open mode as that specified in the d_open call. See the d_open page for information on how the list of database names may be formatted and how databases are located. If any database being opened is a union, then the open mode of the task must be read only.

When multiple databases are being incrementally opened, the databases are assigned sequential numbers from left to right in the order specified in the list, with the first being one greater than the highest previously opened database. The current database will be the first one in the dbnames list upon return from the call. The database number is passed either to function d_setdb or to each database access function, thus setting the current database when multiple databases have been opened. For example, if your application opens two accounting databases named AR_DB and AP_DB, their database numbers would be assigned as follows in the table below.

Database Name Database Number
AR_DB 0
AP_DB 1

The current database is 0.

Then you might incrementally open two additional databases, PR_DB and GL_DB, resulting in the following assignments.

Database Name Database Number
AR_DB 0
AP_DB 1
PR_DB 2
GL_DB 3

The current database is 2.

After closing the database AP_DB using d_iclose, the database numbers would be coalesced as shown below.

Database Name Database Number
AR_DB 0
PR_DB 1
GL_DB 2

If the current database was still 2 before the d_iclose, the current database is now 1, representing the same database, PR_DB.

Your application must track the coalesced database numbers in your application. The d_dbnum function is helpful for this task.

Currency Changes

curr_rec = system record (or NULL_DBA, if no system record);

curr_own[system owned sets] = system record;

curr_own[all other sets] = NULL_DBA;

curr_mem[of all sets] = NULL_DBA;

curr_db = dbn of first database in dbnames;

Example

d_open("sales;invntory", "x", task);
	...

/* eliminate sales and add payroll */
d_iclose(task, 0);
d_iopen("payroll", task);
   	...

/* close all databases */
d_close(task);
