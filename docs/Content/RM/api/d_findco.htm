d_findco

Find current owner of current record

Prototype

int32_t d_findco(
    int32_t  SET, 
    DB_TASK *task, 
    int32_t  dbn)

Parameters

SET (input) The set type for which the owner is to be found.
task (input) A pointer to a DB_TASK structure.
dbn (input) The database number.

Description

This function finds the owner in SET of the current record. The current member of SET will be set from the current record. Both the current owner of SET and the current record are set to the found owner record.

Note: If you want to verify whether the current record is connected to SET, call function d_ismember first to check. Then, if it is connected, call d_findco to retrieve the owner. This will prevent the S_NOTCON error code from being returned.

Currency Changes

curr_own[SET] = current record's owner through SET;

curr_mem[SET] = curr_rec;

curr_rec = current record's owner through SET;

Locking Requirements

A lock on SET or on owner record type of current record is required unless the database is opened in exclusive access mode or opened with the standalone transactional file server (TFS_LIB_TYPE_STANDALONE).

Return Codes

Value Name Description
0 S_OKAY normal return, okay
-2 S_INVSET invalid set
-7 S_NOCR no current record
-16 S_NOTCON record not connected to set
-19 S_INVMEM record not legal member of set
-24 S_NOTLOCKED attempt to access unlocked set/record

Example

/* print keywords associated with tech. info */
while ( d_findnm(INFO_TO_KEY, task, CURR_DB) == S_OKAY )
{
    d_findco(KEY_TO_INFO, task, CURR_DB);
    d_crread(KWORD, key, task, CURR_DB);
    printf("%s\n", key);
}
