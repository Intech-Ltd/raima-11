d_opentask

Allocate memory for a new database task

Prototype

int32_t d_opentask(
    DB_TASK **task);

Parameters

task (output) A pointer to receive a DB_TASK structure pointer.

Description

This function allocates memory for and initializes a database task. It is necessary to obtain a task pointer before calling any other runtime function, as the majority of runtime functions require an initialized task pointer.

A task provides the context, or state, of a set of one or more open databases, and of one transaction that may be actively changing the open databases.

Multiple tasks may be initialized and used in the same program by calling d_opentask multiple times.

Each task represents one "user" in RDM's view. The number of active users in a system is the sum of all open tasks in all active processes. Within one process which has multiple tasks open, the same database(s) may be open in different tasks, and are treated as multiple users, requiring locks to read or update data, and keeping separate page caches.

A task cannot be shared by two concurrent threads in a process, unless those threads synchronize access to the task. However, the runtime functions are reentrant, which means that different concurrent threads can perform runtime operations with different tasks without danger of data corruption.

Currency Changes

None.

Locking Requirements

None.

See Also

d_closetask

Example

DB_TASK *LocalDbTask, *RemoteDbTask;

if (d_opentask(&LocalDbTask) == S_OKAY) {
    if (d_open("tempdb", "x", LocalDbTask) == S_OKAY) {
        if (d_opentask(&RemoteDbTask) == S_OKAY) {
            if (d_open("maindb@ourhq.com", "s", RemoteDbTask) == S_OKAY) {
                /* various database access using both DB_TASKs */
                ...
                d_close(RemoteDbTask);
            }
            d_closetask(RemoteDbTask);
        }
        d_close(LocalDbTask);
    }
    d_closetask(LocalDbTask);
}
