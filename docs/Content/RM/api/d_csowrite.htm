d_csowrite

Write to field in current owner

Prototype

int32_t d_csowrite(
    int32_t     SET, 
    int32_t     FIELD, 
    const void *value, 
    DB_TASK    *task, 
    int32_t     dbn);

Parameters

SET (input) The set type of the current owner to be written.
FIELD (input) The field type from the current owner.
value (input) A pointer to the new value.
task (input) A pointer to a DB_TASK structure.
dbn (input) The database number.

Description

Function d_csowrite writes the data, to which value points to, into data field FIELD in the current owner of SET. If FIELD is a key field, the key is automatically updated. If FIELD is used as a sort field of an ascending or descending ordered set, the record's position in the set is automatically adjusted. If FIELD is an array field, the entire array is written. Elements of an array cannot be written individually. This function cannot be used to modify fields of a compound key. Call d_recwrite to modify compound keys.

Currency Changes

None.

Locking Requirements

Write lock on owner record type, or write lock on SET and key file containing FIELD.

Return Codes

Value Name Description
0 S_OKAY normal return, okay
3 S_DUPLICATE duplicate key
-2 S_INVSET invalid set
-8 S_NOCO set has no current owner
-15 S_ISCOMKEY field is a compound key
-24 S_NOTLOCKED attempt to access unlocked set/record
-27 S_TRNOTACT transaction not active
-91 S_IN_INIT_TRANS Changes not allowed when an initialization transaction is active

Example
