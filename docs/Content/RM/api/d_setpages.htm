d_setpages

Set the number of virtual memory pages

Prototype

int32_t d_setpages(
    int32_t  dbpages, 
    int32_t  ovpages, 
    DB_TASK *task);

Parameters

dbpages (input) The number of cache pages to pre-allocate for the database.
ovpages (input) unused
task (input) A pointer to a DB_TASK structure.

Description

Function d_setpages is used to specify to the RDM runtime system how many pages to pre-allocate for the database cache when the database is opened. The default value is 100 pages. This function must be called prior to d_open. Cache pages will be added as needed if cache page requirements exceed the existing number of allocated pages.

The ovpages value is not used in RDM versions 10 and after, but exists in the function to maintain backward compatibility.

If you do not have enough memory to support the specified number of pages, function d_open will return status S_NOMEMORY, in which case you should reduce the specified amount and try again.

The minimum number for the cache pages is 5.

Currency Changes

None.

Locking Requirements

None.

Return Codes

Value Name Description
0 S_OKAY normal return, okay
-20 S_SETPAGES must call d_setpages() before opening the database
-40 S_INVNUM invalid record or set number

Example
