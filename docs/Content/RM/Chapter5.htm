Summary Listing of Platform Support Functions (psp_ Prefix)

The RDM Platform Support (PSP) provides a rich, platform-independent function interface to sophisticated operating system tasks, such as thread management, concurrency control, dynamic memory allocation, and file management. This section contains detailed descriptions of the RDM PSP functions.

These functions require the header file "psp.h" included in source files. To link these functions, the library name 'psp' must be included (see Library Naming Conventions for library naming conventions).

PSP Return Codes

Value Name Description
0 PSP_OKAY A normal return; successful.
1 PSP_TIMEOUT The event has not been signaled before the time of the timeout value has passed.
2 PSP_FAILED  
3 PSP_NOMEMORY Attempt to allocate memory failed.
4 PSP_NOSERVER Mirroring server not running.
5 PSP_DUPUSERID  
6 PSP_DUPLICATE Database already being mirrored on this slave.
7 PSP_NOSOURCE No source file found in file compare function call.
8 PSP_NODEST No destination file found in file compare function call.
