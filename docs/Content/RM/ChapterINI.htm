Runtime Configuration Settings

Aspects of the runtime environment of RDM can be configured using a configuration file, or programmatically through the API. The TFS environment can be configured using configuration files, through environment variables, or on utility command-lines. Variables set in the environment take precedence over variables set in the configuration file variables set programmatically, or on a command-line will take the highest precedence.

The Runtime configuration encompasses the application program's environment, as distinct from the TFS environment, which controls the actual database files.

Environment Variables

If the IGNOREENV option is programmatically set using d_on_opt, the DBINI and DBUSERID environment variables will be ignored.

Variable Name Description
DBINI Location of rdm.ini file, other than current directory. See d_dbini.
DBUSERID Specific DBUSERID, overrides automatically assigned DBUSERID. See d_dbuserid.
RDM_DOCROOT In the Direct-Link configuration, use this directory instead of current directory for document root.
RDM_TFSERVER Use this domain name instead of localhost to connect to the TFS.

Configuration File (rdm.ini)

The RDM configuration file follows the standard formatting for INI text files. Section names will be enclosed in brackets. Section names and variable names will be case insensitive and comments can be inserted on lines beginning with a semicolon (;).

Unrecognized section names and variables as well as variables appearing in the wrong section names will be ignored by the system. Only one section, [rdm], is used by RDM.

Additional sections and variables can be added to the configuration file to support the user application. RDM includes platform independent support for reading the configuration file. It is recommended that the application developer create a unique section for their application variables rather than reusing one of the above recognized sections to prevent duplicate names if additional RDM variables are added.

Function Description
psp_iniString Retrieve the string definition of a variable in a given section
psp_iniInt32 Retrieve the numeric definition of a variable in a given section
psp_iniSet Set a key/value pair

The currently supported variables and associated section names are located in the table below.

Variable Name Section Name Description
dchainuse rdm See DCHAINUSE
ignorecase rdm See IGNORECASE
readonly rdm See READONLY
timeout rdm See d_timeout
maxcachepages rdm See d_setpages
prealloc_cache rdm See PREALLOC_CACHE

Runtime Options

Option Default State Req'd Database State Description
DCHAINUSE ON Closed Ability to turn off the use of delete chain slots when new records are created.
IGNORECASE OFF Closed Case sensitivity in key files and sorted sets.
READONLY OFF Closed Opening of the database and all database files in read-only mode. The primary use is to facilitate database on CD-ROM devices.
TIMEOUT 10 sec N/A Time, in seconds, for an application to wait for a lock to be granted. If set to 0, application will wait indefinitely.
IGNOREENV OFF Closed Disregard environment variables. The purpose is to allow the application to use the options in the rdm.ini or from the API functions only, regardless of what is in the environment of the user. This option cannot be set in the rdm.ini file, because the environment has already been read when the rdm.ini file is processed.
PREALLOC_CACHE OFF Closed Preallocate the cache pages to ensure there is enough memory.
